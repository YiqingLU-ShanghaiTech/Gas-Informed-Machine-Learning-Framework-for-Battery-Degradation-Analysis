# Gas-Informed Machine Learning Framework for Battery Degradation Analysis

## Project Overview

This repository contains the code implementation for the paper **"Gas-Informed Machine Learning Framework for Stage Classification and Early Forecasting of Battery Degradation"** submitted to ACS Applied Materials & Interfaces. The project leverages operando gas evolution data measured by online electrochemical mass spectrometry (OEMS) to enable earlier degradation-stage identification and long-horizon capacity forecasting for batteries.

## Key Features

- **Stage Classification**: A label-free PCA-Kmeans approach to distinguish between linear and accelerated nonlinear degradation stages
- **Capacity Forecasting**: A single-shot forecaster that fuses short capacity/SOH history with OEMS-measured cycle data
- **Early Detection**: Identifies nonlinear transition 32.0 ± 17.2 cycles before the Bacon-Watts knee (18.0 ± 8.1% of lifetime)
- **Improved Accuracy**: Achieves 1.6–2.4% RMSE at 100-cycle horizons and reduces long-horizon drift compared to capacity-only baselines

## Data Description

- **Capacity Data**: Located in `dataset/CellRetention.xlsx`, contains capacity retention data for all battery cells
- **Gas Data**: Preprocessed gas data is stored in the `processed_data/` folder as `.mat` files
- **Combined Dataset**: `dataset/BatteryDataset.mat` is generated by running the `A01_DatasetOrganization.mlx` script and contains both capacity and gas data for each battery cell
- **Normalized Gas Data**: `dataset/NormalizedGasData.xlsx` provides normalized gas evolution data

## Workflow

### 1. Data Preparation (MATLAB)

1. Run `A00_CellRetention.mlx` to process raw battery data
2. Run `A01_DatasetOrganization.mlx` to organize data into `BatteryDataset.mat`
3. Use supporting MATLAB scripts (`Normalization.m`, `OutliersRemoval.m`, etc.) for data preprocessing

### 2. Stage Classification (Python)

1. Run `C01_KmeansClustering.py` to perform PCA-Kmeans clustering
   - Uses Cell11-1, Cell14, Cell15, and Cell16 as the fixed training set
   - Applies the learned PCA transformation and cluster centers to all remaining test cells

### 3. Capacity Forecasting (Python)

1. Run `F01_BatteryDataProcessor.py` to process battery data for modeling
2. Run `F02_LSTMPredictor.py` to train and evaluate the LSTM baseline model
3. Run `F03_HybridPredictor.py` to train and evaluate the hybrid CNN-LSTM model with cross-attention

### 4. Result Analysis (MATLAB)

1. Run `P01_ResultAnalysis.mlx` to generate prediction visualizations and calculate model errors
   - Results are saved to the `test_results/` folder

## Model Details

### Classification Model
- **Approach**: PCA-Kmeans clustering (unsupervised learning)
- **Input**: OEMS-measured gas evolution profiles (CO, CO2, C2H4)
- **Output**: Classification of degradation stage (linear vs. nonlinear)
- **Performance**: 3.6% misclassification rate (5/136 OEMS-measured cycles)

### Forecasting Models

1. **LSTM Baseline**: Uses only historical capacity data
2. **Hybrid CNN-LSTM**: Fuses 1DCNN-extracted gas features with historical capacity data using cross-attention
3. **Output**: Predicted future capacity trajectories (one-shot prediction without autoregressive rollout)

## Usage

1. **Data Preparation**:
   - Run MATLAB live scripts in alphabetical order (A00, A01)
   - Ensure `dataset/CellRetention.xlsx` and `processed_data/` files are present

2. **Stage Classification**:
   ```bash
   python C01_KmeansClustering.py
   ```

3. **Capacity Forecasting**:
   ```bash
   python F01_BatteryDataProcessor.py
   python F02_LSTMPredictor.py
   python F03_HybridPredictor.py
   ```

4. **Result Analysis**:
   - Run `P01_ResultAnalysis.mlx` in MATLAB
